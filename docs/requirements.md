# 要件定義書

## プロダクト概要

### 名称

**AI Diary Companion** - 会話から自動生成される日記サービス

### プロダクトコンセプト

- **会話ベースの日記生成**: ユーザーが「日記を書く」という意識なく、AIとの日常会話から自動的に日記が生成される
- **リアルな人間との会話体験**: 即座に返信が来るのではなく、性格や職業、時間帯に応じた自然な返信タイミングを実現
- **柔軟なAIキャラクター設定**: 性別、関係性、性格、職業などを設定可能な複数のAIキャラクターが会話相手として機能
- **継続しやすい日記習慣**: 「何を書けばいいかわからない」「続かない」という従来の日記の課題を、自然な会話で解決

### プロダクトビジョン

日記をつけたいが続かない人、話し相手が欲しい人に対して、AIとの自然な会話を通じて自動的に日記が生成される体験を提供します。ユーザーは「日記を書く」というタスクから解放され、日常の会話を楽しむだけで記録が残る世界を実現します。将来的には、LINE/Slack/Discordなどの既存チャットプラットフォームと連携し、ユーザーが普段使っているツールでシームレスに利用できるようになります。

### 目的

- 日記を書くことのハードルを下げ、継続的な記録を支援する
- AIとの会話を通じて、ユーザーの日常に寄り添う体験を提供する
- 拡張しやすいアーキテクチャで、プロトタイプから本格的なサービスへの発展を可能にする

## ターゲットユーザー

### プライマリーペルソナ: 田中さくら（28歳、会社員）

- 毎日日記をつけたいと思っているが、何を書けばいいかわからず続かない
- 仕事や日常生活のストレスがあり、誰かに話を聞いてほしいと感じることがある
- スマートフォンでLINEやSlackを日常的に使用している
- 現在の課題: 日記アプリをダウンロードしても3日で挫折する、書くネタが思いつかない
- 期待する解決策: 自然に会話するだけで日記が残る、プレッシャーなく続けられる仕組み
- 典型的なワークフロー: 朝起きてスマホをチェック、通勤中にSNSを見る、帰宅後にリラックスしながらチャットで友人と会話

### セカンダリーペルソナ: 山田健太（35歳、フリーランスエンジニア）

- 在宅勤務で一人で作業することが多く、話し相手が欲しい
- 自分の考えや進捗を整理するために、定期的に振り返りをしたい
- プライマリーとの違い: 日記そのものより、思考整理やメンタルヘルスのサポートが主な関心事

## 成功指標(KPI)

- 日次会話頻度: 1日あたり5通以上のメッセージ交換を週5日以上継続するユーザーが50%以上（プロトタイプリリース後3ヶ月以内）
- 日記生成数: ユーザーあたり月間20件以上の日記が自動生成される（プロトタイプリリース後3ヶ月以内）
- ユーザー満足度: 「会話が楽しい」「日記が続けられそう」と感じるユーザーが80%以上（ユーザーアンケートで測定）

## 機能要件

### コア機能(MVP)

#### チャットインターフェース

**ユーザーストーリー**:
ユーザーとして、AIキャラクターと自然に会話するために、チャットインターフェースが欲しい

**受け入れ条件**:

- [ ] ユーザーがメッセージを入力し送信できる
- [ ] AIキャラクターからの応答がリアルタイムに表示される
- [ ] 会話履歴が時系列で表示される
- [ ] モバイル/デスクトップの両方で利用可能

**優先度**: P0(必須)

#### AI会話生成機能

**ユーザーストーリー**:
ユーザーとして、自然な会話体験を得るために、AIが文脈を理解して応答する機能が欲しい

**受け入れ条件**:

- [ ] ユーザーの入力に対して自然な応答が生成される
- [ ] 会話の文脈（過去の発言）を考慮した応答ができる
- [ ] 応答生成時間が5秒以内（P95）
- [ ] 不適切な応答（暴力的、差別的な内容）をフィルタリングする

**優先度**: P0(必須)

#### AIキャラクター設定機能

**ユーザーストーリー**:
サービス管理者として、ユーザーに多様な会話体験を提供するために、AIキャラクターの設定を柔軟に変更できる機能が欲しい

**受け入れ条件**:

- [ ] キャラクター名、性別、性格、関係性（友人/恋人/家族など）を設定できる
- [ ] 職業（会社員/学生/フリーランス/主婦など）を設定できる
- [ ] 生活パターン（勤務時間帯、休日など）を設定できる
- [ ] 設定したキャラクター属性が会話の口調や内容に反映される
- [ ] 複数のキャラクタープリセットを作成・管理できる
- [ ] 新しいキャラクターをコード変更なしで追加できる（パラメータ注入方式）

**優先度**: P0(必須)

#### プッシュ通知機能

**ユーザーストーリー**:
ユーザーとして、アプリを閉じていてもAIからの返信を受け取るために、プッシュ通知が欲しい

**受け入れ条件**:

- [ ] AIからの返信が届いたとき、アプリを閉じていてもプッシュ通知が届く
- [ ] 通知をタップするとチャット画面に遷移する

**優先度**: P0(必須)

---

#### AI応答制御機能（リアルな返信タイミング）

**ユーザーストーリー**:
サービス管理者として、普通の人間と会話しているかのような自然な体験を提供するために、キャラクターの性格・職業・時間帯に応じた返信タイミングを制御できる機能が欲しい

**受け入れ条件**:

- [ ] 性格による返信速度パターンを設定できる
  - マメなキャラクター：数分程度で返信
  - 普通のキャラクター：数十分〜1時間程度で返信
  - 忙しい/マイペースなキャラクター：数時間後に返信
- [ ] 職業・生活パターンによる返信時間帯を制御できる
  - 会社員：平日9-18時は返信が遅い、昼休み(12-13時)や夜(19時以降)は返信が早い
  - 学生：平日9-15時は返信が遅い、放課後(16時以降)は返信が早い
  - フリーランス：比較的自由だが、作業中は返信が遅い
- [ ] 返信頻度を設定できる（必ず返信 / 時々既読スルー / 応答率○%など）
- [ ] ランダム性を持たせて、毎回同じタイミングにならないようにする
- [ ] ユーザーが長時間会話している場合、AIから話題を振る機能がある
- [ ] 設定した応答パターンが実際の会話に反映される

**優先度**: P0(必須)

#### 日記自動生成機能

**ユーザーストーリー**:
ユーザーとして、会話内容を振り返るために、1日の会話から自動的に日記が生成される機能が欲しい

**受け入れ条件**:

- [ ] 1日の会話内容をもとに、要約された日記テキストが生成される
- [ ] 日記は日付ごとに整理される
- [ ] 生成された日記をユーザーが確認・閲覧できる
- [ ] 日記生成のタイミングは1日1回（深夜0時など）

**優先度**: P0(必須)

#### 日記閲覧機能

**ユーザーストーリー**:
ユーザーとして、過去の記録を振り返るために、生成された日記を日付別に閲覧できる機能が欲しい

**受け入れ条件**:

- [ ] 日付ごとの日記一覧が表示される
- [ ] 特定の日付の日記詳細を開いて読める
- [ ] カレンダー形式で日記がある日を視覚的に確認できる
- [ ] 過去30日分の日記に素早くアクセスできる

**優先度**: P1(重要)

#### ユーザー認証機能

**ユーザーストーリー**:
ユーザーとして、自分の日記データを保護するために、ログイン機能が欲しい

**受け入れ条件**:

- [ ] メールアドレスとパスワードでアカウント登録できる
- [ ] 登録済みアカウントでログインできる
- [ ] ログアウト機能が提供される
- [ ] パスワードはハッシュ化して保存される

**優先度**: P1(重要)

### 将来的な機能(Post-MVP)

#### LINE/Slack/Discord連携

ユーザーが普段使っているチャットプラットフォーム上でAI Diary Companionを利用できるようにする。Webhook/Bot APIを使った連携機能。

**優先度**: P2(できれば)

#### 日記の検索機能

キーワードや期間で過去の日記を検索できる機能。

**優先度**: P2(できれば)

#### 感情分析

会話内容や日記から感情を分析し、ユーザーのメンタル状態の変化を可視化する。

**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス

- チャット画面の初期表示時間: 2秒以内
- AI応答生成の内部処理時間: 30秒以内（ユーザーには見えない。実際の返信タイミングはキャラクター設定に依存）
- 日記生成処理時間: 特に制限なし（バッチ処理で非同期実行）

### ユーザビリティ

- 新規ユーザーがアカウント登録から最初の会話まで3分以内に到達できる
- モバイル・デスクトップの両方でレスポンシブに動作する
- 直感的なUIで、説明なしに基本操作ができる

### 信頼性

- データ損失ゼロ（会話履歴・生成された日記は永続化される）
- サービス稼働率: 95%以上（プロトタイプ期間中）

### セキュリティ

- パスワードはbcryptなどでハッシュ化して保存
- 通信はHTTPSで暗号化
- ユーザーの日記データは他のユーザーから参照できない（アクセス制御）

### スケーラビリティ

- 初期フェーズでは同時接続ユーザー数100人程度を想定
- データベース設計は将来的な水平スケーリングを考慮

## スコープ外

明示的にスコープ外とする項目:

- **外部チャットプラットフォーム連携（LINE/Slack/Discord）**: MVP完成後に別途試作
- **日記の手動編集機能**: 自動生成された日記をユーザーが編集する機能は初期バージョンでは提供しない
- **音声入力/音声応答**: テキストベースの会話のみをサポート
- **マルチ言語対応**: 初期は日本語のみ
- **有料課金機能**: プロトタイプでは無料で提供
- **ソーシャル機能（日記の共有など）**: 個人利用に特化し、SNS的な要素は含めない
