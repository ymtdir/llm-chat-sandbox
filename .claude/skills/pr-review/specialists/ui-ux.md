# UI/UX

UI/UXの一貫性・アクセシビリティ・ユーザー体験を評価します。

## デザインの基本原則: C.R.A.P.

| 原則                         | 確認ポイント                                                                         |
| ---------------------------- | ------------------------------------------------------------------------------------ |
| **Contrast（コントラスト）** | 重要な要素が視覚的に目立っているか。主要アクションと副次的アクションが区別できるか   |
| **Repetition（反復）**       | フォント・ボタンスタイル・カラーが一貫しているか。統一感のあるデザインか             |
| **Alignment（整列）**        | 要素が意図的に配置されているか。テキスト・ボタン・セクションがグリッドに揃っているか |
| **Proximity（近接）**        | 関連する要素がグループ化されているか。無関係な要素は適切に離れているか               |

## ニールセンの10ヒューリスティクス

レビュー時にUIの変更が以下の原則に反していないかを確認する。

1. **システム状態の可視性**: 処理中・成功・失敗がリアルタイムにフィードバックされるか
2. **実世界との一致**: ユーザーが理解できる言葉・アイコンを使っているか
3. **ユーザーの制御と自由**: 取り消し・やり直しが容易か。誤操作から簡単に復帰できるか
4. **一貫性と標準**: 同じアクションが常に同じ見た目・振る舞いをするか
5. **エラーの予防**: 破壊的操作に確認ダイアログがあるか。そもそもミスしにくい設計か
6. **記憶より認識**: 選択肢が見えているか。ユーザーに暗記を求めていないか
7. **柔軟性と効率性**: パワーユーザー向けのショートカットがあるか
8. **美的で最小限のデザイン**: 不要な情報がなく、本質に集中できるか
9. **エラーの認識と回復**: エラーメッセージが平易な言葉で、解決方法を示しているか
10. **ヘルプ**: 必要に応じてタスク指向のヘルプが提供されているか

## アクセシビリティ: WCAG 2.1

### レベルA（最低限）

- 画像・メディアに代替テキスト（alt属性）があるか
- 色だけに依存せず、形状やテキストでも情報を伝えているか
- すべての機能がキーボードで操作できるか

### レベルAA（標準）

- テキストのコントラスト比が4.5:1以上、グラフィックは3:1以上か
- フォーカスインジケーターがキーボードユーザーに見えるか
- テキストが200%まで拡大しても機能が損なわれないか

## よくあるUIアンチパターン

以下のパターンを検出した場合は指摘する。

- **不統一なスペーシング**: padding/marginが不規則でグリッドが崩れている
- **ローディング状態の欠如**: ボタン押下後にフィードバックがない。遅い接続でEmpty stateが一瞬表示される
- **不親切なエラーメッセージ**: 「エラーが発生しました」のみで具体的な対処法がない
- **フォーカス状態の不可視**: キーボードユーザーが現在位置を把握できない
- **説明なしのdisabled**: なぜ操作できないのかユーザーに伝わらない
- **確認なしの破壊操作**: 削除・送信に確認ダイアログがない
- **ラベルなしのフォーム**: label要素が欠落、またはplaceholderだけに依存している

## チェック項目

- [ ] C.R.A.P.原則に反するUI変更がないか
- [ ] UIコンポーネントの使い方がプロジェクトの慣習と一貫しているか
- [ ] レスポンシブ対応が適切か
- [ ] WCAG 2.1のレベルA・AAを満たしているか
- [ ] ユーザー操作に対するフィードバック（ローディング、エラー表示等）があるか

## レビュー手順

1. 変更ファイルのdiffからUI関連の変更を特定する
2. C.R.A.P.原則とニールセンのヒューリスティクスに照らして問題を検出する
3. アクセシビリティ（WCAG）のチェックポイントを確認する
4. 上記のアンチパターンに該当しないか確認する

## 指摘の観点

- **一貫性の欠如**: 既存のUIパターンやデザインシステムと異なるスタイル
- **アクセシビリティ不足**: aria属性の欠落、キーボード操作不可、不十分なコントラスト
- **フィードバックの欠如**: ローディング状態やエラー状態の表示がない
- **レスポンシブ未対応**: 特定の画面サイズで表示が崩れる可能性がある実装
