# セキュリティ

セキュリティリスクや脆弱性の有無を検証します。

## OWASP Top 10 (2021)

レビュー時に以下の脅威カテゴリを念頭に置く。

| #   | カテゴリ                 | 概要                                               |
| --- | ------------------------ | -------------------------------------------------- |
| A01 | アクセス制御の不備       | 権限チェックの欠落・バイパスによる不正アクセス     |
| A02 | 暗号化の失敗             | 弱い暗号化・平文での機密データ保存・転送           |
| A03 | インジェクション         | ユーザー入力がSQL・コマンド・LDAPとして解釈される  |
| A04 | 安全でない設計           | 脅威モデリングの欠如・設計レベルのセキュリティ不備 |
| A05 | セキュリティ設定ミス     | デフォルト設定・不要サービスの公開・未パッチ       |
| A06 | 脆弱な依存コンポーネント | 既知のCVEを持つライブラリの使用                    |
| A07 | 認証の不備               | 弱い認証情報管理・セッション管理の欠陥             |
| A08 | データ整合性の不備       | 安全でないデシリアライズ・未検証の更新             |
| A09 | ログ・監視の不備         | 不十分な監査証跡・アラート欠如                     |
| A10 | SSRF                     | アプリが未検証のURLへリクエストを送信              |

## セキュアコーディング原則

- **最小権限**: 必要最小限のアクセス権のみ付与する
- **多層防御**: 複数の独立したセキュリティ制御を重ねる。1つが破られても他で防ぐ
- **フェイルセーフ**: デフォルトでアクセス拒否。ホワイトリスト方式で許可する
- **完全な仲介**: エントリポイントだけでなく、すべてのリクエストで権限チェックする

## 認証・認可のチェックポイント

### 認証

- セッショントークンは暗号的に安全な乱数（64bit以上のエントロピー）を使用しているか
- ログイン後にセッションIDを再生成しているか（セッション固定攻撃の防止）
- CookieにSecure・HttpOnly・SameSite=Strict属性が設定されているか
- セッションタイムアウト（絶対: 2-8h、アイドル: 15-30min）が設定されているか

### 認可

- すべてのリクエストで認可チェックが行われているか（デフォルト拒否）
- リソースの所有者検証がされているか（水平権限昇格の防止）
- URLパラメータやCookieの操作でバイパスできないか

### ログ

- 認証の試行（ユーザー、タイムスタンプ、成否）をログに記録しているか
- パスワード・トークン・個人情報はログに出力していないか

## 主要な脆弱性パターン

| パターン                 | 確認ポイント                                                 |
| ------------------------ | ------------------------------------------------------------ |
| SQLインジェクション      | パラメータ化クエリ・プリペアドステートメントを使用しているか |
| XSS                      | 出力エンコード・入力バリデーション・CSPヘッダーがあるか      |
| CSRF                     | SameSite Cookie・CSRFトークン・Origin検証があるか            |
| SSRF                     | 許可リストによるURL制限・内部IPアクセスの禁止があるか        |
| 安全でないデシリアライズ | 信頼できないデータのデシリアライズを避けているか             |

## チェック項目

- [ ] 入力値のバリデーションが適切か（OWASP A03）
- [ ] 認証・認可の処理が上記チェックポイントを満たしているか
- [ ] 機密情報がハードコード・ログ出力されていないか
- [ ] 依存パッケージに既知の脆弱性がないか
- [ ] エラーメッセージに内部情報（スタックトレース、DB構造等）が露出していないか

## レビュー手順

1. 変更ファイルのdiffからセキュリティに関連する箇所を特定する
2. OWASP Top 10の脅威カテゴリに照らして問題を検出する
3. 認証・認可のチェックポイントを確認する
4. ハードコードされた秘密情報がないかをスキャンする

## 指摘の観点

- **入力検証の欠如**: ユーザー入力が未検証のままDBやシステムに渡されている
- **認可バイパス**: 権限チェックが不十分で他ユーザーのリソースにアクセス可能
- **機密情報の露出**: シークレットのハードコード、ログへの機密情報出力
- **安全でない依存**: 脆弱性が報告されているライブラリの使用
