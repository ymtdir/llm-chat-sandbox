# 基本設計書作成スキル

高品質な基本設計書を作成するためのスキルです。要件定義書の要件を技術的に実現するためのシステム構造・テクノロジースタックを定義します。

## 前提条件

- **必須**: `docs/requirements.md`（要件定義書）が作成されている

## 出力ルール

`../../rules/document-generation-guide.md` を参照。

**対象ファイル**: `docs/basic-design.md`

## 参照ファイル

- **テンプレート**: `./assets/template.md`
- **作成ガイド**: `./references/guide.md`

## 基本設計書作成プロセス

### 1. 入力の確認

- `docs/requirements.md` を読み、プロダクト概要・機能要件・非機能要件を把握する
- 既存の `docs/basic-design.md` があるか確認。ある場合は `../../rules/document-generation-guide.md` の「既存ドキュメント最優先」ルールに従い、既存の節構造・表記・言語を維持したまま更新する
- `docs/requirements.md` が存在しない場合は下記トラブルシューティングに従う（skill を中止）

### 2. 基本設計書ドラフトの生成

`docs/requirements.md` の内容をもとに、テンプレート（`./assets/template.md`）に従って基本設計書を生成します。詳細な記述ポイントは `./references/guide.md` を参照してください。

行数の目安は **300-400 行以内（上限目安、簡潔優先）**。超過する場合は `../../rules/document-generation-guide.md` に従いサブディレクトリ（`docs/design/`）に分割する。

### 3. 基本設計書のレビューと改善

生成された基本設計書を以下 5 観点でレビューし、問題があれば修正する:

1. アーキテクチャ選定の妥当性（選定理由が明確か）
2. データ設計の整合性（要件の主要エンティティを網羅しているか）
3. セキュリティ網羅性（認証・認可・暗号化・機密情報管理）
4. 非機能要件の達成手段（パフォーマンス・稼働率・スケーラビリティ）
5. 要件トレーサビリティ（各機能・非機能要件が設計に反映されているか）

**終了条件**: 品質チェックリスト（本 skill 末尾）の全項目が ○ になり、かつユーザー（人間）が承認した時点で終了する。

**レビュー結果の扱い**: レビュー結果・改善ログは `docs/basic-design.md` 本体には含めず、会話上のログとしてユーザーに提示する（本体は成果物として簡潔に保つ）。

**注意点**: AI のレビューを鵜呑みにせず、最終的な判断は必ず人間が行う

## 既存ドキュメント更新時の整合判断

既存の `docs/basic-design.md` があり、新しい要件が既存設計と関係する場合:

- **要件に既存設計の廃止が明示されている**: 置換（既存記述を削除し新設計で差し替え）
- **廃止記述がない**: 併存（既存を残し、新設計を追記して関係を注記）
- **スタイル整合**: 既存が英語節・箇条書き主体なら新規追記もそれに合わせる（template の日本語節構造を強制しない）

## 品質チェックリスト

### テクノロジースタック

- [ ] 言語・ランタイム・フレームワーク・ライブラリ・開発ツールが明記されているか
- [ ] 各技術選定の理由が記載されているか

### システムアーキテクチャ

- [ ] アーキテクチャパターンの選定理由が明確か
- [ ] レイヤー/コンポーネント間の依存関係が定義されているか
- [ ] アーキテクチャ図（Mermaid 等）が含まれているか

### データ設計

- [ ] データモデルが図で表現されているか
- [ ] ストレージ方式と選定理由が記載されているか
- [ ] バックアップ戦略が定義されているか

### セキュリティ

- [ ] データ保護（暗号化・アクセス制御）が記載されているか
- [ ] 入力検証・認証方式が明記されているか
- [ ] 機密情報の管理方針が記載されているか

### 非機能要件

- [ ] パフォーマンス要件の達成手段が具体的に記載されているか
- [ ] 稼働率・スケーラビリティ要件への対応が記載されているか

## トラブルシューティング

- **前提ドキュメントが不足**: `docs/requirements.md` が必要です。`requirements-definition` skill で作成してから再実行してください。
- **既存の設計書との競合**: `../../rules/document-generation-guide.md` の優先順位ルールに従ってください（既存ドキュメント最優先）。
