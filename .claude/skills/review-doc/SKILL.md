# ドキュメントレビュースキル

ドキュメントの品質を評価し、改善提案を行うためのスキルです。

## 呼び出しモード

本 skill は 2 つの文脈で呼ばれる:

1. **ユーザー直接呼び出し**（skill を直接起動した場合）: 「直接呼び出し時の手順」に従い、引数検証 → `doc-reviewer` agent 起動 → 結果要約を行う
2. **doc-reviewer agent 内部**（agent からロードされた場合）: 「レビュープロセス」節以降に従って実際のレビューを実行する

呼び出し元が agent 自身の場合は再度 agent を起動しない（無限ループ防止）。agent は自身の入力契約からそれを判断する。

## 入力契約

- **ドキュメントパス**: 必須（例: `docs/requirements.md`）
- 引数が無い場合は以下を返して終了:

  ```
  引数が指定されていません。

  使い方: 「<ドキュメントパス> をレビューして」のようにパスを指定してください。
  例: docs/requirements.md をレビューして
  ```

## 直接呼び出し時の手順

### ステップ1: 引数バリデーションと存在確認

1. ドキュメントパスを取得する。引数が無ければ「入力契約」のメッセージを返して終了
2. 指定されたパスの存在を確認する。存在しない場合は「エラー応答」のフォーマットを返して終了（agent は起動しない）

### ステップ2: doc-reviewer サブエージェント起動

Agent tool を使用して doc-reviewer を起動する:

- subagent_type: `doc-reviewer`
- prompt: `<ドキュメントパス> をレビューしてください。`

### ステップ3: レビュー結果の要約

doc-reviewer の返却（出力契約）を以下のフォーマットで要約してユーザーに報告する:

```
✅ <ドキュメントパス> のレビューが完了しました。

【総合スコア】: <N>/5
【評価サマリ】:
  - 完全性 / 明確性 / 一貫性 / 実装可能性 / 測定可能性 の ✅/⚠️/❌
【必須対応】: <件数> 件

詳細は doc-reviewer の返却レポートを参照してください。
```

## レビュー観点

1. **完全性**: 必要な情報が全て含まれているか
2. **明確性**: 曖昧な表現がなく、誰が読んでも理解できるか
3. **一貫性**: 文書内および関連ドキュメントと矛盾がないか
4. **実装可能性**: 読み手が次のアクションを取れるレベルか
5. **測定可能性**: 基準や要件に具体的な数値があるか（該当する場合）

## レビュープロセス

1. 対象ドキュメントを読み込む（存在しない場合は下記「エラー応答」に従う）
2. ドキュメントの種類・目的を特定する
3. 対応するスキルやテンプレートがあれば参照する（下記「対応スキルのマッピング」参照）
4. `./references/review-guide.md` に従ってレビューする
5. 出力フォーマットに従って結果を出力する

### 対応スキルのマッピング

| 対象ファイル | 参照する作成 skill |
|---|---|
| `docs/requirements.md` | `requirements-definition` |
| `docs/basic-design.md` | `basic-design` |
| `docs/detailed-design.md` | `detailed-design` |
| `docs/development-guide.md` | `development-guide` |

該当 skill の「品質チェックリスト」を併せてレビュー観点とする。マッピングに無いドキュメントは `review-guide.md` のみで評価。

### エラー応答

対象ドキュメントが存在しない場合はレビューを強行せず、以下の応答を返す:

```
⚠️ レビュー対象 `<パス>` が見つかりません。

確認事項:
- パス指定が正しいか
- ファイルがまだ未作成か

正しいパスを指定して再実行してください。
```

## 評価基準

### スコアと評価アイコンの対応

| スコア | 評価アイコン | 意味 |
|---|---|---|
| 1-2 | ❌ | 重大な問題あり |
| 3 | ⚠️ | 軽微な改善点あり |
| 4-5 | ✅ | 問題なし |

### 総合スコアの算出

該当する観点の算術平均。「測定可能性」など該当しない観点がある場合は除外して平均する（該当するか否かは対象ドキュメント種別により判断）。
